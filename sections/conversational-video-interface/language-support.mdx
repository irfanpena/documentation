Tavus supports a wide range of languages for spoken interaction, powered by third-party text-to-speech (TTS) engines. The available languages depend on the TTS engine selected for the conversation.

## Default Behavior

By default, Tavus uses the **Cartesia** TTS engine, which supports many global languages.

## Setting the Conversation Language

To specify a language, use the `language` parameter in the request body. **You must use the full language name**, not a language code.

```shell cURL highlight={9}
curl --request POST \
  --url https://tavusapi.com/v2/conversations \
  --header 'Content-Type: application/json' \
  --header 'x-api-key: <your_api_key>' \
  --data '{
  "persona_id": "pdced222244b",
  "replica_id": "rfe12d8b9597",
  "properties": {
    "language": "spanish"
   }
}'
```

<Note>
Language names must match exactly with those supported by the selected TTS engine.
</Note>

## Multilingual Mode

```shell cURL highlight={9}
curl --request POST \
  --url https://tavusapi.com/v2/conversations \
  --header 'Content-Type: application/json' \
  --header 'x-api-key: <your_api_key>' \
  --data '{
  "persona_id": "pdced222244b",
  "replica_id": "rfe12d8b9597",
  "properties": {
    "language": "multilingual"
   }
}'
```

This enables ASR (Automatic Speech Recognition) to detect and lock in the spoken language for the duration of the conversation.
